# 同花顺模拟盘 · 每周Top5操作指南

## 一、快速开始

```bash
# 进入项目目录
cd D:\opencode工作区\deep_quant

# 激活虚拟环境
.venv\Scripts\activate

# 运行虚拟盘交易（自动完成: 数据加载→模型训练→选股→输出报告）
python virtual_trade.py
```

- **飞书推送** — 交易指令推送到飞书群

## 二、核心参数配置

编辑 `virtual_trade.py` 顶部（第 23-32 行）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `INITIAL_CAPITAL` | `200000` | 总资金（元） |
| `TOP_K` | `5` | 最终持仓股票数 |
| `MIN_LOTS` | `1` | 每只至少买 N 手（1手=100股） |
| `CANDIDATE_POOL` | `15` | 候选池：从 Top N 中筛选 |
| `COMMISSION` | `0.00025` | 佣金（万2.5） |
| `STAMP_DUTY` | `0.001` | 印花税（千1，卖出） |
| `PRICE_TOLERANCE` | `0.02` | 挂单缓冲（现价+2%确保成交） |

**调整示例：**

```python
# 改为 100万资金，选 Top 3，每只至少买 2手
INITIAL_CAPITAL = 1000000
TOP_K = 3
MIN_LOTS = 2
CANDIDATE_POOL = 10
```

## 三、可购买性筛选原理

```
         Top15候选池（按预测得分排序）
                  │
         ┌────────┴────────┐
         ▼                 ▼
     [选中] 京东方A     [替换] 宁德时代
     价格 4.50              价格 520.00
     ≈20%预算=40K          ≈20%预算=40K
     可买 8800股(88手) ✓   只能买 0手 ✗
         │                 
         ▼                 ▼
     进入持仓             跳过，尝试 Top6 → Top7 ...
```

**替换逻辑：**
1. 从 `CANDIDATE_POOL`（默认 Top15）开始
2. 按预测得分从高到低逐只检查
3. 判断 `预算(约20%) ≥ 1手价格 × 100股 × (1+费率)`
4. 通过 → 入选；不通过 → 自动尝试下一候选
5. 重复直到凑满 `TOP_K` 只可购买股票

**常见被替换场景：** 高价股如宁德时代(>400元)、比亚迪(>200元) 在 20万资金下可能无法买满1手，会自动替换为预测得分次高的中低价股。

## 五、每周执行流程

建议 **每周一开盘前** 运行：

```
周一 09:00   →  python virtual_trade.py
             →  打开 output/thss_weekly_picks_周一.txt
             →  按清单在同花顺模拟盘逐只买入
             →  剩余资金留仓，等待下周调仓
```

也可设置 Windows 任务计划程序自动运行。

## 六、参数调优建议

| 场景 | 建议配置 |
|------|----------|
| 资金少（5-10万） | `CANDIDATE_POOL=20` 扩大候选池，更多低价股替换机会 |
| 资金多（100万+） | `MIN_LOTS=5` 每只至少买5手，`CANDIDATE_POOL=10` |
| 想分散风险 | `TOP_K=8` 持仓增至8只 |
| 想集中持仓 | `TOP_K=3` 仅选3只 |
| 高价股多（当前市场） | `MIN_LOTS=1` `CANDIDATE_POOL=15` 积极替换 |
| 低价股多（市场下跌后） | `MIN_LOTS=3` 充分利用资金 |

## 七、常见问题

**Q: 选不够5只股票怎么办？**
A: 放宽约束从剩余池补充。极端情况下若所有候选都无法购买，会输出少于5只的持仓并提示。

**Q: 如何切换股票池？**
A: 修改 `config.py` 中的 `SECTOR_MAP` 和 `STOCK_POOL`。

**Q: 报告中的"挂单价"高于现价？**
A: `PRICE_TOLERANCE=0.02` 表示高于现价2%挂单，确保在模拟盘中优先成交，实际成本按现价计算。
